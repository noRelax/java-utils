MetricsAnalyzer: Project Status

This is the first alpha release of MetricsAnalyzer. The following are known deficiencies, and will probably be addressed in the future:

  • The JavaNCSS importer tries to skip metrics related to inner classes, because the JavaNCSS output for inner classes is ambiquous (two distinct inner classes may have the exact same name). It can easily be formally proven, that this problem in JavaNCSS output can not be entirely solved by MetricsAnalyzer (without the possiblity of associating the metrics incorrectly).
  • There is insufficient documentation at the moment.
  • Query visualization and export is missing.
  • There is no way to remap the relations produced by Importers. This makes it more difficult for the user to combine the outputs of third party Importers (in case someone ever writes those).
  • The code that collects metrics from the Importers and stores them to the database is probably rather slow at the moment. The code should be profiled and optimized.

Contributions, such as

  • constructive feedback,
  • importers,
  • queries,
  • documentation,
  • architecture (design & implementation) for query visualization and export) and
  • success stories,

will be appreciated and acknowledged! CVS write access is generally granted to developers who are willing to integrate and maintain major contributions to MetricsAnalyzer.
